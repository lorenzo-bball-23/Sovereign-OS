### 1. The Command Center (Home Screen)
* **The Priority Stack:** Positioned dead center. It displays exactly **three concrete actions** prioritized dynamically by the backend based on your latest mental friction logs.
* **The Metrics Rail:** Displays your rolling *Sovereign Ratio* (Clean vs. Dirty Fuel allocation) and your *Resistance Baseline* trend graph.
* **Central Input Router:** A unified input engine at the bottom center. You log raw data, text snippets, or audio transcripts and route them immediately to their designated processing module.

### 2. The Homework Lab (The ERP Engine)
* **The Mechanic:** Houses behavioral disruption tasks (Exposure and Response Prevention) designed to trigger and dismantle specific avoidance patterns, such as the reflexive need to hide actions or appear perfectly productive.
* **The Audit Flow:** Checking off a task requires completing a 3-point modal tracking system:
  1. **Resistance Rating (RR):** A 1–5 slider measuring localized neural friction.
  2. **Motive Tagging:** Binary identification of whether the action was driven by the *Vision* or by *Image Protection*.
  3. **3-Second Rule Verification:** Automated compliance tracking regarding immediate execution parameters.

### 3. The Vault (Threaded Topical Diaries)
* **The Sandbox Protocol:** To prevent cognitive bloat and choice paralysis, the diary is organized into completely isolated conversational threads. 
* **Core Memory Integration:** While text chats remain visually separated for absolute focus, underlying insights update your central profile database via the demo schema parser.
* **Anti-Stall Logic:** The AI automatically identifies *Relief-Seeking* (venting for a dopamine hit) vs. *Resolution-Seeking*. If an unproductive loop is flagged, the thread locks and demands a physical action from the user before unlocking.

### 4. The Relationship Module (Siloed Social Audit)
* **The Rationale:** Romantic and social interactions require a ring-fenced testing environment to keep attachment dynamics, deactivation strategies, or emotional projections from contaminating professional performance datasets.
* **The Mechanic:** Employs Attachment Theory to parse social friction and cross-references data against personal history layers, generating targeted behavioral experiments for field settings.

## 🛠️ Technical Specifications & Cost Optimization

To manage resource consumption and safeguard against high API overhead within European data frameworks, the platform employs a dual-stage local architecture:

* **The Framework:** Built on full-stack **Next.js** for high-performance server-side state processing, seamless data routing, and native responsive rendering on mobile viewports.
* **Dual-Model Routing:** Daily transactional logs, quick inputs, and automated text routing utilize **Gemini 1.5 Flash** for low latency and near-zero cost. Intensive structural analysis, weekly scorecards, and Monthly Founder Integrity Audits automatically route to **Gemini 1.5 Pro**.
* **Context Caching Engine:** Core diagnostic and therapy files are cached directly via **Vertex AI** servers. Instead of paying to re-upload your entire profile context with every single transaction, the application reads the cached server state, reducing active token overhead by roughly **90%**.
* **Hard Quota Governance:** The linked Google Cloud billing project is hard-capped with an absolute monthly spend limit and strict minute-by-minute rate-limiting to prevent structural application errors from incurring unmonitored charges.
